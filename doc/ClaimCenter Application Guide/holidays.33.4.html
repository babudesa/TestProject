Using Gosu Methods to Work with Holidays
You can write Gosu code to set business days differently for various tasks. For example:
•
A regulatory requirement mandates that a task be completed within a defined number of business days. Your code can takes into account the holiday schedule of an agency in a certain jurisdiction.
•
After auto-assigning a task to be completed in a certain number of business days, Gosu code can take into account the holiday schedule of the assignee.
•
Gosu code can check General holiday types in all zones through which the mail passes to determine the correct number of days to allow for mail to be delivered. Use this code for determining when to send time sensitive mail.
Use Gosu methods which use Holiday Type and Zone to determine the correct number of business days.
Gosu Holiday Methods that Use Zones and Types
These methods compute business days taking into consideration holiday types or zones. These methods are defined in gw.util.GWBaseDateEnhancement.
 
Methods that use Holidays
This is what they do:
getConfiguredHolidays()
Return the array of all holidays of all types and zones.
getConfiguredHolidays(tagCode : HolidayTagCode)
Return the array of all holidays of this type.
getConfiguredHolidays(location : Zone)
Return the array of all holidays in this zone.
addBusinessDays(date : Date, iDays : int, tagCode : HolidayTagCode)
Adds int (signed) business days to Date, based on holidays of the specified type. Time of day is preserved.
addBusinessDays(date : Date, iDays : int, location : Zone)
Adds int (signed) business days to Date, based on holidays in the specified zone. Time of day is preserved.
businessDaysBetween(fromDate : Date, toDate : Date, tagCode : HolidayTagCode)
Returns number of business days between the given dates, omitting holidays of the specified type; ignores hours in dates.
businessDaysBetween(fromDate : Date, toDate : Date, location : Zone)
Returns number of business days between the given dates, omitting holidays in the specified area; ignores hours in dates.
