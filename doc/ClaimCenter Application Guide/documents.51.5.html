Document Management Integration
The following are the main interfaces used to integrate to a document management system.
 
Interface
Description
IDocumentMetadataSource
ClaimCenter passes search parameters (metadata) to this interface, which searches its metadata and returns a list of documents found.
•
Interface to a system for storing document metadata (name, id, status, author, and so on). If the system specifies none, then the ClaimCenter database stores the metadata.
•
Separate from IDocumentContentSource because of different architectural requirements.
IDocumentContentSource
ClaimCenter passes this interface metadata for one file, which returns its content.
•
Interface to a document storage system.
•
Contains methods for creating, updating, and retrieving document contents.
•
Supports several document retrieval modes:
•
Document contents.
•
Web page containing an ActiveX control (see “Working With Documents”).
•
Gosu executed by client rules.
•
URL to a local content store.
IDocumentProduction
•
Interface to a document creation system.
•
Document creation process can involve extended workflow and/or asynchronous processes.
•
Can depend on or set document fields.
IDocumentTemplateSource and IDocumentTemplateDescriptor
•
Basic interfaces for searching for and retrieving templates describing the document to be created.
•
Includes basic metadata (name, MIME type, and so on) and a pointer to the template content.
For import details about document management and related integration points, see “Document Management” in the Integration Guide.
