Subrogation Data Model

Values

The following entities support subrogation:
Entity
Description
Claim
There are three fields on Claim that are related to subrogation:
•
SubrogationStatus – Type key to SubrogationStatus
•
SubrogationSummary – Foreign key to SubrogationSummary
•
subrogator – Derived property returning Contact, the external subrogation firm for the claim
Matter
There is a field on Matter that is related to subrogation:
•
SubroRelated – Boolean indicating if the matter has a related subrogation
StatuteLimitationsLine
Represents a statute of limitations for a subrogation. The field SubrogationSummary is a foreign key to the associated SubrogationSummary entity. There is also a derived key for the associated claim.
The SubrogationSummary entity has an array key, StatuteLine, to support multiple statute limitations.
SubroAdverseParty
Stores subrogation related information for a third party who is the subject of a subrogation recovery for a claim. This entity does not represent the third party’s insurance company. This entity has a derived field for the associated claim, and it has foreign keys to Contact and SubrogationSummary.
The field SubroAdverseParty on SubroPaymentSchedule is a foreign key to this entity. Additionally, the SubrogationSummary entity has an array key, SubroAdverseParties, to support multiple adverse parties.
SubrogationSummary
Represents a subrogation for a claim. An object of this type is instantiated for every new claim.
SubroPaymentSchedule
Represents a promissory note schedule for an adverse party who is the subject of a subrogation. The field SubroAdverseParty is a foreign key to the associated SubroAdverseParty entity.
