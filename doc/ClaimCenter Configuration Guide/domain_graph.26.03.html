ClaimCenter Configuration Guide : Data Model Configuration : The Domain Graph : What is the Admin Graph?
What is the Admin Graph?
Guidewire defines admin data as data that ClaimCenter shares with multiple data objects rather than with a single instance of an object. This is, for example, data the many claim objects share such as catastrophe data. Some examples of this type of data includes the following:
•
Activity patterns
•
Catastrophes
•
Groups
•
Question Sets
•
Regions
•
Users
The admin graph represents this type of data.
Admin Graph Object Characteristics
An admin graph represents the set of entities that have all of the following characteristics:
•
Entities that are not part of the domain graph.
•
Entities that more than one domain graph instance share.
•
Entities that must exist in the archive database in order to preserve foreign key relationship, for example, User and Group objects. Thus, because the User object is part of the admin graph, so is the Credential object. This is necessary because of the foreign key relationship between the two objects.
The admin graph is not necessarily a DAG (Directed Acyclic Graph). It can be a partitioned graph.
Admin Graph Synchronization
ClaimCenter does not archive the admin graph. Instead, ClaimCenter synchronizes the admin graph with the main application database. In other words, ClaimCenter mirrors the admin graph from the main database into the archive database. This means the table rows in the primary archive database and the main application database for the admin entities are identical except for the object ID.
The archive process uses the following tables to keep track of the mapping of the entity ID in the main database to the entity ID in the primary archive database.
•
ArchiveAdminKey table
•
ArchiveTypeKey table
In the insert-select operation, the archive process uses these tables to determine the value of the ID of the admin entities in the archive database. For example, if User with ID 100 has ID 200 in the archive database, ClaimCenter uses the value 200 in a insert-select operation if the domain graph refers to 100.
IMPORTANT Guidewire recommends that you treat the main application database and the primary archive database as a single linked database. For example, do not drop one and not drop the other. Or, do not swap one for the other. If you do, Guidewire does not guarantee the behavior of the system after this change.
