Searching on Japanese Names
There are many possible readings for Japanese names. Japanese names thus require extra fields in the database to capture the phonetic notation, furigana, in either hiragana or katakana characters. These fields need to be searchable as primary search criteria.
To accomplish this, you need to do the following:
1.
Extend the Person entity and add extra fields for use with the Japanese first name and last names. Set supportsLinguisticSearch equal to true for both of these fields.
2.
Extend the Contact entity and add an extra field for use with Japanese organization names. Set supportsLinguisticSearch equal to true.
3.
Make these fields searchable as Primary Search Criteria.
IMPORTANT If you intend to perform a name search on katakana Japanese names, you must also set supportsLinguisticSearch to true on Contact entities.
The following sections describe the steps necessary to perform a primary search on Japanese names with furigana.
•
Step 1: Extend the Contact Entity
•
Step 2: Enable Search from ClaimCenter to ContactCenter
•
Step 3: Enable Internal ClaimCenter Search
Step 1: Extend the Contact Entity
Perform the following example steps to extend the Contact entity and add additional fields to it that you can use to search on Japanese names.
1.
In ClaimCenter Studio, add the following fields to Contact.etx under Data Model Extensions → extensions for the company name:
 
<column desc="The contact name in Kanji." name="NameK" supportsLinguisticSearch="true" 
        type="companyname"/>
2.
In ClaimCenter Studio, add the following fields to Person.etx for the first and last names. If this file does not exist, then you need to create it under Data Model Extensions → extensions. (Navigate to Data Model Extensions → metadata → pl, right-click Person.eti, and select Create extension file from the menu.)
 
<column desc="Kanji first name." name="FirstNameK" 
        supportsLinguisticSearch="true" type="firstname"/>
 
<column desc="Kanji last name." name="LastNameK" 
        supportsLinguisticSearch="true" type="lastname"/>
3.
In ContactCenter Studio, repeat steps 1 and 2 for ABContact and ABPerson.
If the extension files for these entities do not exist, then you need to create them. For example, if the ABPerson.etx file does not exist, navigate to Data Model Extensions → metadata → ab, right-click ABPerson.eti, and select Create extension file from the menu.
Step 2: Enable Search from ClaimCenter to ContactCenter
After extending the Contact object with additional name fields, perform the following steps to enable the use of these fields for searching in Guidewire ClaimCenter.
1.
In ClaimCenter Studio, add the following fields to the ContactSearchCriteria entity extension to enable searching on the new Contact fields. Adding these fields enables you to have separate search fields for the new last name field and the new first name field.
 
<column desc="Kanji First Name" name="FirstNameK" type="shorttext"/>
<column desc="Kanji Last Name" name="LastNameK" type="shorttext"/>
<column desc="Kanji Company Name" name="CompanyNameK" type="shorttext"/>
2.
In ContactCenter Studio, add the same columns to ABContactSearchCriteria.
3.
In ContactCenter Studio, open search-config.xml (under Other Resources).
a.
Add a new CriteriaDef for ABCompany.
 
  <CriteriaDef entity="ABContactSearchCriteria" targetEntity="ABCompany">
    <Criterion property="CompanyNameK" targetProperty="Name" matchType="startsWith"/>
  </CriteriaDef>
b.
Add the following criteria to the CriteriaDef element <CriteriaDef entity="ABContactSearchCriteria" targetEntity="ABPerson">:
 
  <Criterion property="FirstNameK" matchType="startsWith"/>
  <Criterion property="LastNameK" matchType="startsWith"/>
4.
Restart both ClaimCenter Studio and ContactCenter Studio.
5.
In ContactCenter Studio, open config.xml (in Other Resources) and add the new fields to configuration parameter AdditionalPrimaryABContactSearchCriteria. Adding these fields to this configuration parameter makes it possible to search with only the new fields specified. See “AdditionalPrimaryABContactSearchCriteria” in the Configuration Guide for more information on this configuration parameter.
<param name="AdditionalPrimaryABContactSearchCriteria" value="FirstNameK,LastNameK,CompanyNameK"/>
Note: You can also optionally update the MinimumSearchCriteria display properties so that they give a more correct error message given that there are three more primary search fields.
6. Restart the ContactCenter server
7. In ClaimCenter Studio, refresh the abintegration web service against the running ContactCenter server. (Open Web Services → abintegration and click Refresh.)
6.
In ClaimCenter Studio and ContactCenter Studio, add the new fields to the appropriate search PCF pages. For example, for ClaimCenter:
•
Add the new fields to AddressBookSearchDV.pcf to enable using these fields for searching.
•
Add the new fields to ContactBasicsDV.<mode>.pcf to enable editing these fields in ContactCenter.
7.
In ClaimCenter Studio, if necessary, change the IAddressBookAdapter plugin to use CCAddressBookPlugin. Navigate to Plugins → gw → plugins → addressbook and click IAddressBookAdapter. If the demonstration configuration still exists, remove it and add a new Gosu plugin with the following implementation:
 
Gosu class:
gw.plugin.ccabintegration.impl.CCAddressBookPlugin
username:
ClientAppCC
password:
gw
8.
Start the ClaimCenter application server.
After performing these steps, you can now add a Japanese name with the furigana fields in ContactCenter, then search for it in ClaimCenter. For completeness, you can also change the Company and Person display names to reflect the new fields in some way.
Step 3: Enable Internal ClaimCenter Search
The following steps enable you to search against the ClaimCenter database for some other primary object like Claim and to specify the new contact fields to narrow the claim search. These steps show how to add the fields to a claim search.
If you performed the previous steps, then you have already have done the ClaimCenter portion of extending the Contact object in ClaimCenter and ContactCenter. Perform these steps in ClaimCenter Studio.
1.
Create an extension for CCNameCriteria and add the new Contact fields to it.
 
<column desc="The first name in Kanji, if searching for a person." 
        name="FirstNameK" type="firstname"/>
<column desc="The last name in Kanji, if searching for a person." 
        name="LastNameK" type="lastname"/>
<column desc="The company name in Kanji, if searching for a company." 
        name="CompanyNameK" type="companyname"/>
2.
Open search-config.xml (under Other Resources).
a.
Add to the following to CCNameCriteria → Company mapping:
  <Criterion property="CompanyNameK" targetProperty="Name" matchType="startsWith"/>
b.
Add the following to the CCNameCriteria → Person definition for the new fields:
 
  <Criterion property="FirstNameK" matchType="startsWith"/>
  <Criterion property="LastNameK" matchType="startsWith"/>
3.
Stop and start both ClaimCenter Studio and the ClaimCenter application server.
Again, you need to modify the PCF search screens for your new contact search fields to be visible.
