Understanding Graph Notation
Guidewire uses the DOT plain text graph description language to describe both the domain and admin graphs. The DOT language provides a way of describing a complex graph of relationships using human-readable text, but which specialized software can use to generate a visual representation of the graph. DOT graph files generally end with the .dot extension.
For example, in ClaimCenter, the domain graph representation in the DOT language begins like this:
digraph MyGraph {
    Claim [shape=box,style=filled,color=".7 .3 1.0"];
    Address -> Contact[style=dashed]
    ClassCode -> Policy[arrowtail=crow,arrowhead=onormal]
    EmploymentData -> ClassCode
    PolicyLocation -> Policy[arrowtail=crow,arrowhead=onormal]
    ...

Within the graph, the notation has the following meaning:
•
Arrows point in the direction of foreign keys. The direction of the arrow is the is owned by relationship.
•
Dashed (or dotted) arrows mean that the foreign key is owned by the parent entity, which you indicate by setting the owner attribute on the foreign key to true.
•
A bold arrow from an entity to itself means an edge foreign key.
•
An open arrow from one entity to another entity represents either an array (multi-pronged tail) or a one-to-one relationship (single line tail).
•
Blue entities and links mean that these entities and links were defined through data model extensions.
