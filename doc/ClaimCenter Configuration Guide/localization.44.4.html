Enabling Guidewire Localization Content
In the base configuration, Guidewire provides a limited number of optional non-US localization modules to support application localization for specific countries. You can access these optional modules from the customer portal.
The countries for which Guidewire provides localization modules are:
 
Country
Code
Great Britain
GB
Australia and New Zealand
au-nz
Note: Guidewire bundles country-specific content for Australia and New Zealand into one module package as these two countries share a large number of locale formats.
Each country-specific module provides localization for a single non-US region or geographic area and supports the following:
•
A single base currency
•
A single language, although it is possible to support additional languages for that region
•
A single set of business processes, set of rules, activity patterns, and similar items
•
A single set of business days and hours
In particular, ClaimCenter provides the following localized Administration screens:
•
Catastrophes
•
Regions
•
Holidays
•
Business Week
IMPORTANT A country-specific module does not supply the postal or zone data for that country. Guidewire expects you to provide this type of data.
Directory Structure
Within the application directory structure, you install a country-specific module folder in the following location:
ClaimCenter/modules/...
Guidewire uses the following naming syntax for these modules:
xx-locale-yy
The prefix xx is the standard abbreviation for a Guidewire application. These are:
 
ab
ContactCenter
bc
BillingCenter
cc
ClaimCenter
pc
PolicyCenter
The suffix yy is the standard two-character ISO code for a country, with the exception of the Australia/New Zealand module that uses au-nz as the country code. For example:
cc-locale-GB
cc-locale-au-nz
For a list of valid country codes, see the following web site:
http://www.iso.org/iso/english_country_names_and_code_elements
Enabling a Country-Specific Module
Although Guidewire provides optional country-specific modules in the base configuration, ClaimCenter does not recognize any country-specific module unless you activate it.
IMPORTANT Guidewire strongly recommends that you do not enable more than one country-specific content package at a time.
WARNING Do not modify any file in any country-specific module folder. Doing so can invalidate your ClaimCenter installation and prevent it from starting thereafter.
To activate a country-specific module
IMPORTANT Perform the following steps before you start the application server for the first time.
The following steps apply to the Great Britain module as an example.
1.
Download and install the country-specific module in the correct application folder.
2.
Start Guidewire Studio and open config.xml for editing. (You can search for it in the Other Resources folder, or use CTRL-N and enter the file name to load the file into a Studio view.)
In the base configuration, Guidewire defines a default application database and an archive database. However, you must set configuration parameter ArchiveEnabled to true, before you can start the application server. (Or, you can delete the <archive> element under the default database instead, if you want.)
IMPORTANT Do not modify any file in the country-specific folder. If you need to make changes to config.xml, for example, use Studio to manage this process as it places a copy of config.xml in the correct location.
Each country-specific module contains its own copy of config.xml file. Within config.xml, the following parameters are of particular importance for enabling a country-specific module:
•
DefaultApplicationLocale
•
DefaultCountryCode
Each country-specific module sets these two parameters differently, as needed for that locale.
WARNING It is not possible to change either one of these configuration parameters after you start the database for the first time. Any attempt to do so invalidates your installation.
3.
Open modules.xml for editing. You can find this file in the root of the modules folder in the installation directory.
IMPORTANT The modules.xml file is one of the very rare cases in which you modify an application file outsides of the ClaimCenter/modules/configuration folder. Do not modify any file in the ClaimCenter/modules folder other than modules.xml.
4.
Search for a module containing name=”configuration”. Add a new <dependency> element to this module. You use this dependency to set the country-specific module that you wish to activate. For example, for Great Britain, enter the following:
 
<module name="configuration">
  <dependency type="module">cc</dependency>
  <dependency type="module">cc-locale-GB</dependency>
</module>
IMPORTANT The name that you enter for the <dependency> module must match the name of the installation country-specific folder.
5.
Add a module definition for your country-specific module. For example, for Great Britain, enter the following after the configuration module:
 
<module name="cc-locale-GB">
  <property name="locale">en_GB</property>
  <dependency type="module">cc</dependency>
</module>
6.
(Optional) If you have integrated Guidewire ClaimCenter with Guidewire ContactCenter, then you also need to refresh the abintegration WSDL. Otherwise, you do not need to perform this step.
a.
Open Guidewire Studio and navigate to Resources → Web Services.
b.
Open the abintegration web service.
c.
Click Refresh, located in the upper right-hand corner of the screen.
Note: You must have the ContactCenter (AB) server integrated with the ClaimCenter server and running before you attempt to refresh the WSDL.
7.
Start the ClaimCenter application server. The application server uses the default application locale defined for the country-specific module.
Upgrading a Localization Module
If you are upgrading your application, be sure to review the current ClaimCenter release notes. It is possible for Guidewire to add additional display keys as part of the application upgrade. If so, you must add the new display keys to the display.properties files (including those for Studio and Gosu).
