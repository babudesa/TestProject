Deploying to WebSphere
For WebSphere, ClaimCenter is packaged into a WebSphere-specific EAR file. This topic explains the steps needed to configure WebSphere and deploy an EAR file.
To deploy ClaimCenter on WebSphere, first follow the common instructions that apply to any ClaimCenter instance on WebSphere. See "Common Steps for WebSphere". Then, follow instructions for either production or development environments. See either "Deploying a ClaimCenter Production Environment on WebSphere" or "Deploying a ClaimCenter Development Environment with WebSphere".
Notes:
•
Guidewire only supports externally managed data sources using JDBC drivers shipped with ClaimCenter and not the JDBC drivers that might be installed by default with the application server.
•
Use the correct port number when you connect to the server in Studio. WebSphere’s default port is 9080.
•
Have the correct version of WebSphere. See "Application Server Requirements".
Common Steps for WebSphere
Procedures in this section are common to all deployments of ClaimCenter on WebSphere. These procedures include generating the ClaimCenter EAR file and adding a welcome file list. For any deployment of ClaimCenter on WebSphere, follow the procedures in this section. Then proceed to either "Deploying a ClaimCenter Production Environment on WebSphere" or "Deploying a ClaimCenter Development Environment with WebSphere".
To generate the ClaimCenter EAR File for WebSphere
1.
Open a command window.
2.
Navigate to the ClaimCenter bin directory.
3.
Enter the following command:
gwcc build-websphere-ear
The command builds the EAR file and places it in ClaimCenter\dist\ear.
To add a welcome-file-list
This section explains how to add a wecome-file-list to the ClaimCenter web.xml file. If you do not perform this step, WebSphere can not find your index.html file. Pointing your browser to: http://localhost:9080/cc does not work. Point to: http://localhost:9080/cc/Start.do.
1.
Open the ClaimCenter EAR file with a utility such as WinZip.
2.
Open the ClaimCenter WAR file within the ClaimCenter EAR file.
3.
Open the WEB-INF\web.xml file for editing.
4.
At the bottom of the file, just before the </web-app> tag, add the following:
 
<welcome-file-list>
  <welcome-file>index.html</welcome-file>
  <welcome-file>index.htm</welcome-file>
  <welcome-file>index.jsp</welcome-file>
</welcome-file-list>
5.
Save your changes and close web.xml. Your extraction utility might prompt you to update the ClaimCenter WAR file with the updated web.xml. Update the ClaimCenter WAR file to include your updated web.xml. The utility probably also prompts you to replace the repackaged WAR file within the ClaimCenter EAR file, since the WAR file contents have changed.
Note: The gwcc build-websphere-ear command does not package the files in webapps\WEB-INF when rebuilding an EAR file. If you later need to repackage the EAR file, follow this procedure again.
Deploying a ClaimCenter Production Environment on WebSphere
To deploy a production instance of ClaimCenter on WebSphere, first generate the ClaimCenter EAR file and add a welcome file list. Follow the instructions in "Common Steps for WebSphere". Then, use the following procedure to install the EAR file on WebSphere.
To install the ClaimCenter EAR file on WebSphere for a production environment
1.
If WebSphere is not already running, start the application server.
2.
Open the WebSphere Administrative Console.
3.
From the left menu, expand Applications.
4.
Click New Application.
5.
Select New Enterprise Application.
6.
Click Browse and select the ClaimCenter EAR file in the ClaimCenter\dist\ear directory.
7.
Click Next.
8.
Select Fast Path.
9.
Click Next.
10.
Accept the default installation options.
11.
Click Next.
12.
On the Map modules to servers page, verify that your targeted server or cluster is selected.
13.
Click Next.
14.
On the Map virtual hosts for Web modules page, verify that default_host is selected for the Virtual host.
15.
Click Next.
16.
On the Summary page, review your selections for accuracy. Click Previous to change any settings.
17.
Click Finish.
18.
Click Save to apply the changes to the master configuration.
19.
Restart WebSphere.
After you have finished the procedure to install the ClaimCenter EAR file on WebSphere, proceed to "Starting ClaimCenter on WebSphere".
Deploying a ClaimCenter Development Environment with WebSphere
To deploy a development instance of ClaimCenter on WebSphere, first generate the ClaimCenter EAR file and add a welcome file list. Follow the instructions in "Common Steps for WebSphere". Then, follow the procedures in this section. These procedures enables you to configure ClaimCenter without requiring you to repackage the ClaimCenter EAR file. When you have finished these procedures, stop and restart WebSphere.
If you decide to use WebSphere for a development environment, you create and deploy a ClaimCenter EAR file the first time you set up ClaimCenter, but not for subsequent changes. You only need to regenerate the ClaimCenter EAR file when you are ready to migrate to a different server for your production environment.
To install the ClaimCenter EAR file on WebSphere for a development environment
1.
If WebSphere is not already running, start the application server.
2.
Open the WebSphere Administrative Console.
3.
From the left menu, expand Applications.
4.
Click New Application.
5.
Select New Enterprise Application.
6.
Click Browse and select the ClaimCenter EAR file in the ClaimCenter\dist\ear directory.
7.
Click Next.
8.
Select Fast Path.
9.
Click Next.
10.
Accept the default installation options, except for the following:
•
Select Use Binary Configuration.
•
Select Enable Class Reloading.
•
Select Deploy Web Services.
•
For Validate Input off/warn/fail, select off.
•
Select Process Embedded Configuration.
11.
Click Next.
12.
On the Map modules to servers page, verify that your targeted server or cluster is selected.
13.
Click Next.
14.
On the Map virtual hosts for Web modules page, verify that default_host is selected for the Virtual host.
15.
Click Next.
16.
On the Summary page, review your selections for accuracy. Click Previous to change any settings.
17.
Click Finish.
18.
Click Save to apply the changes to the master configuration.
19.
Restart WebSphere.
To copy necessary WebSphere files to the ClaimCenter webapps directory
1.
Navigate to the installed web application at
WAS HOME\profiles\profile\installedApps\cell\cc.ear
For example: 
C:\Program Files\IBM\WebSphere\AppServer\profiles\
AppSrv01\installedApps\<COMPUTERNAME>Node01Cell\cc.ear
2.
Copy the META-INF directory to the ClaimCenter webapps directory.
For example:
ClaimCenter\webapps
If you do not see this directory, restart WebSphere.
3.
Navigate to the cc.war directory inside the exploded ClaimCenter EAR directory within WebSphere.
WAS HOME\profiles\profile\installedApps\cell\cc.ear\cc.war
4.
Copy the META-INF directory from cc.war to the ClaimCenter webapps\cc directory.
For example:
ClaimCenter\webapps\cc
5.
Navigate to the WEB-INF directory inside the cc.war directory within WebSphere.
For example:
WAS HOME\profiles\profile\installedApps\cell\cc.ear\cc.war\WEB-INF
6.
Copy the files ibm-web-ext.xmi and ibm-web-bnd.xmi and paste them to the ClaimCenter webapps\cc\WEB-INF directory.
For example:
ClaimCenter\webapps\cc\WEB-INF
To edit deployment.xml for WebSphere
This section describes how to edit the deployment.xml file in the WebSphere deployed application location to point back to your original web application installation.
1.
Navigate to:
 
WAS HOME\AppServer\profiles\profile\config\cells\
cell\applications\cc.ear\deployments\cc
For example, your path could be:
C:\Program Files\IBM\WebSphere\AppServer\profiles\AppSrv01\config\cells
\cell\applications\cc.ear\deployments\cc
2.
Open deployment.xml for editing.
3.
On line 3, find binariesURL= and change the entry to the ClaimCenter webapps directory, such as:
ClaimCenter\webapps
4.
On line 6 find uri= and change the entry from cc.war to cc.
5.
Save and close deployment.xml.
To edit application.xml for WebSphere
This section describes how to edit the application.xml file in your original web application.
1.
Navigate to the META-INF directory that you copied to the ClaimCenter webapps directory. For example:
ClaimCenter\webapps\META-INF
2.
Open application.xml for editing.
3.
On line 10, change the web-uri entry from cc.war to cc.
4.
Save and close application.xml.
After you have finished this procedure, proceed to "Starting ClaimCenter on WebSphere".
