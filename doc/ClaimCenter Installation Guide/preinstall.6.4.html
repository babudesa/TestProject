ClaimCenter Installation Guide : Before Installing ClaimCenter : Installing Java
Installing Java
The ClaimCenter application server and Guidewire Studio require a JVM (Java Virtual Machine). The version of the JVM depends on the servlet container and operating system on which the application server runs.
 
Component
Java installation
Tomcat on Windows
ClaimCenter requires the Sun JRE 6. (Java 1.6.0 or higher.)
WebSphere
Use the IBM JDK included with WebSphere.
WebLogic
Use the Sun JDK included with WebLogic.
Configuration environment, administration tools
Sun Java SDK 1.6.0_21 or higher, 32-bit
You can only build using the Sun SDK.
Guidewire Studio
Sun JDK 1.6.0 or higher, 32-bit
IMPORTANT Production environments must use a 64-bit operating system and 64-bit JVM. Development environments must use a 32-bit JVM.
By default, Sun JVMs provide both a client and a server mode. Guidewire supports only the server mode as it yields much higher performance. How you set this mode depends on your application server.
•
If using Sun JVM with Tomcat, then add the -server flag to CATALINA_OPTS.
•
If using Sun JVM with WebLogic, then add the -server flag as an argument while launching the WebLogic start script.
•
If using the IBM JVM with WebSphere, the server mode is enabled by default. You probably do not need to change any settings.
Refer to http://www.oracle.com/technetwork/java/javase/downloads/index.html for information on downloading JDK 6.
