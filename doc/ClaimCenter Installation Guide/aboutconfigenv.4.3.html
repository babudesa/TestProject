Edited Resource Files Reside Only in the Configuration Module
The configuration directory is the only place for user-edited resources (an example of the path might be: c:/ClaimCenter/modules/configuration/config). ClaimCenter runs a checksum process upon startup that verifies that no files have been changed in any directory except for those in the configuration directory. If this process detects an invalid checksum, the application does not start. In this case, revert any changes to all modules except for the /configuration directory and try again.
Guidewire recommends that you create a read-only backup copy of this directory available as you develop. This enables you to easily recover from accidental changes that would prevent the application from starting up.
