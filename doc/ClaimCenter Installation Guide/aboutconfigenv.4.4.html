If You Accidentally Edit Files Outside of the Configuration Directory
In the event that files in any module except in /modules/configuration/ are edited in place, you are unable to start Guidewire Studio or ClaimCenter. If this happens, you see an error like this during the verify-checksum routine, directly after entering the command to start:
verify-checksum:
     [java] Verifying module checksum: cc
     [java] invalid aggregate checksum
     [java] file failed checksum: config/contact/entityroleconstraints-config.xsd
     [java] Verifying module checksum: cc-toolkit
     [java] Verifying module checksum: core
     [java] Verifying module checksum: integration
     [java] Verifying module checksum: pl-activex
     [java] Verifying module checksum: pl-test
     [java] Verifying module checksum: pl-toolkit
     [java] Verifying module checksum: platform
     [java] Verifying module checksum: plintegration
BUILD FAILED
The checksum verifies that all files, except those designated to be user-edited, have not changed, and if they have, it stops Studio and the application from running.
To correct this, you first need to know what module contains the problem file. The error message lists the file name, including the path to the file. Duplicate configuration resources are sometimes stored in many directories in a single installation of the Guidewire application.
To find out which module holds this particular file instance, look at the directory that verify-checksum was checking as the error occurred. In the previous error message example, the first module, cc, failed the checksum because it contained a file that had been mistakenly edited.
In this example, to fix the checksum error, copy the entityroleconstraints-config.xsd file from:
backup/ClaimCenter/modules/cc/config/contact
and copy to:
ClaimCenter/modules/cc/config/contact
Result: The restored file enables the checksum routine to complete successfully, so the application can start.
