Gosu Composition Syntax
Gosu classes now support a composition syntax. This allows a Gosu class to delegate an interface to another class. In other words, another class implements the interface methods and your class calls the delegate class to handle those methods. You define the delegate in the class definition and reference the delegate through an internal class field. This compositional model allows for easy implementation of objects that are proxies for other objects.
For example, the following Gosu class MyStringList defines a delegate for all methods on List<String>. That means that other code can interact with a MyStringList class as if it directly handles the methods. However, this class delegates the details to another object, in this case an instance of ArrayList<String>.
class MyStringList implements List<String> 
{
  private delegate _internalList represents List<String> = new ArrayList<String>()
}
Other code can now use an instance of MyStringList and call ArrayList methods that the delegate handles. 
var x = new MyStringList()
x.add( "TestString" ) // this is a method on ArrayList, but we call the method on our custom class
For more information, see “Composition” in the Gosu Reference Guide.
