ClaimCenter Integration Guide : Web Services (SOAP) : Built-in Web Services
Built-in Web Services
ClaimCenter API functions are grouped by purpose within various interfaces within the package com.guidewire.cc.webservices.api. The following table summarizes these interfaces:
 
Interface
Description
General web services
ISystemToolsAPI
This interface provides a set of tools that are always available, even if the server is set to DBMaintenance run level. One of its most important tools is getting and setting the server’s run level. This particular API provides the current server and schema versions. See “System Tools Web Services”
IMaintenanceToolsAPI
This interface provides a set of tools that are available only if the system is at Maintenance run level or higher. This particular API can start various background processes. You must poll the server later to see if the process failed or completed successfully. See “Maintenance Web Services”.
IImportTools
Imports system data from an XML file. You must only use this with administrative database tables (entities such as User). This system does not perform complete data validation tests on any other type of imported data. See “Importing Administrative Data”.
IMessagingToolsAPI
Various messaging methods. For example, retry a message or get the counts of pending or failed messages for an external system. See “Messaging and Events”.
ITypelistToolsAPI
Tools for getting valid typecodes for a typelist and mapping between ClaimCenter internal codes and the codes for external systems. See “Mapping Typecodes to External System Codes”.
IUserAPI
Manipulates user data. See “User and Group Web Services”.
IGroupAPI
Manipulates group data. See “User and Group Web Services”.
ITableImportAPI
Table-based import interface for high volume data import. This would be used typically for large-scale data conversions, particularly if migrating claims from a legacy system into ClaimCenter prior to bringing ClaimCenter into production. See “Importing from Database Staging Tables”.
ITemplateToolsAPI
This interface includes methods that list the available Gosu templates and validate the syntax of the templates. See “Data Extraction Integration”.
WorkflowAPI
Designate a workflow to immediately check if it can advance to its next step. See “Workflow Web Services”.
ILoginAPI
Log into the server if you use Microsoft .NET or other languages other than Gosu and Java 1.5. See “Calling Your Web Service from Microsoft .NET (WSE 3.0)” for how to use it with .NET. In general, do not use this for Java development as it is better to use the APILocator Java class. For Java login, see “Calling Your Web Service from Java”. If you use more than one Guidewire application, do not use this to connect from Gosu language to the other server. For more information, see “Testing Your Web Service”.
IProfilerAPI
Sends information to the built-in system profiler. For more information, see “Profiling Web Services”.
ZoneImportAPI
Takes geographic zone data in comma separated value (CSV) format and populates staging tables with the data, in preparation for importing the data. You probably do not need to use web service directly. Instead, use the command line tool zone_import, which provides a simple interface for importing your CSV data that is already stored locally. See “Importing from Database Staging Tables”.
 
 
ContactCenter-related web services
IContactAutoSyncAPI
Address books use this web service for automatic contact autosync functionality. If a contact updates in an address book, the address book notifies ClaimCenter to pick up changes. ClaimCenter finds contacts to synchronize to new values in the external address book. If you use ContactCenter and have it installed, ContactCenter uses this automatically. See “Address Book Integration”.
 
 
Claim and exposure web services
IClaimAPI
Manipulates claims and claim data. Methods add documents to a claim, create a new claim or FNOL (first notice of loss), and add activities to claims using an activity pattern as a template. See “Claim APIs”.
IClaimFinancialsAPI
Manipulates financials information associated with claims. Methods perform actions such as loading reserves, payments, and recoveries from an external system, or update status of a payment. See “Claim Financials Web Services”.
IExposureAPI
Various actions on exposures, such s adding documents and notes, reopening an exposure, and getting an exposure’s status. See “Exposure APIs”.
ISREEAuthenticationAPI
This is for Guidewire use only. This is part of the internal implementation to communicate between ClaimCenter and the optional reporting module for ClaimCenter. Do not directly use this web service interface. For more information about reporting, see ClaimCenter Reporting Guide
The full specifications of the web service API interfaces and their methods are documented in the SOAP API Reference Javadoc documentation. For more information about developer documentation, see “Integration Documentation Overview”.
