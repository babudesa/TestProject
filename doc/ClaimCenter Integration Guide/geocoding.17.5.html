ClaimCenter Integration Guide : Geographic Data Integration : Geocoding Status Codes
Geocoding Status Codes
Geocoding service providers typically provide a set of geocoding status codes that indicate what happened during the geocoding attempt. For example, even if latitude and longitude coordinates successfully return, it is useful to know whether those coordinates represent. Do they represent an exact address match? If the service could not find the address, did it return the coordinates for the weighted center of the postal code? Additionally, status codes that indicate complete geocoding failure or whether an address was ever geocoded.
The status values must be values from the GeocodeStatus typelist, described in the following table:
 
Geocoding status code
Description
NONE
No attempt at geocoding this address ever occurred. This is the default geocoding status for an address. Do not set an address to this geocoding status. If you experience an error that must retrigger geocoding later, throw an exception instead.
FAILURE
An attempt at geocoding this address was made but failed completely. If an address is not geocodeable, use this code. The distributed work queue that attempts background geocoding continually (and infinitely) retries the failed address. Do not use this code for an error that is retryable. If you experience an error that must retrigger geocoding later, throw an exception instead.
EXACT
This address was geocoded successfully and the service provider indicates that the supplied geocode is an exact match for the address.
STREET
This address was geocoded successfully and the service provider indicates that the supplied geocode is an exact match for the street (not the complete address). This can be more or less specific than postal code, depending on the exact address and how physically long a given street is.
POSTALCODE
The returned coordinates are based on the postal code.The meaning of a postal code match depends on the geocoding service. A geocoding service may use the weighted center of the geographic area, or may use a specific/defined address for either such as a post office or city hall.
CITY
Similar to POSTALCODE, but the returned coordinates are based on the cityâ€™s weighted center. The meaning of a CITY match depends on the geocoding service. A geocoding service may use the weighted center of the geographic area, or may use a specific/defined address for either such as a city hall.
IMPORTANT The GeocodeStatus typelist listed earlier is final. This typelist is not extendible.
