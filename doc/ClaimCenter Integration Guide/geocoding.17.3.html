Deploying a Geocode Plugin Overview
There are several high-level steps for deploying geocoding:
Step 1: Implement the plugin interface in Gosu
If you want to support another geocoding service, write your own geocoding plugin and register it with the system. See "Writing a Geocoding Plugin".
If you want to support proximity searches of vendors within the ContactCenter application, you must repeat this in ContactCenter’s version of Studio.
IMPORTANT If you use proximity searching of vendors, remember to repeat this for ContactCenter.
Step 2: Register the Plugin
Within Studio, register your new plugin in the plugins editor, choose the GeocodePlugin interface for the plugin interface name. In Studio under Resources, right-click on Plugins. In the dialog that appears, choose the GeocodePlugin interface. In the right-pane, click Add... and select Gosu from the picker.
If you want to pass additional configuration parameters such as a user name or password to the plugin, you can do configure that in the plugins editor. See “The Messaging Editor” in the Configuration Guide for details.
If you support proximity searches of vendors (which necessarily uses the ContactCenter application), you must repeat this step in ContactCenter’s version of Studio.
IMPORTANT If you use proximity searching of vendors, remember to repeat this for ContactCenter.
If you are using the built-in MapPoint implementation, you must also set up the web service in Studio. For more information, see “Using the Geocoding Feature” in the System Administration Guide.
Step 3: Enable User Interface for Desired Features
You also need to enable the user interface for the desired features within ClaimCenter with configuration parameters in the config.xml file.
For ClaimCenter, modify the UseGeocodingInPrimaryApp parameter. This specifies whether ClaimCenter displays the user interface for proximity searches local to ClaimCenter in assignment and user search pages/pickers. For example:
<param name="UseGeocodingInPrimaryApp" value="true"/>
If you support proximity searches of vendors (which uses the ContactCenter application) and want to support the user interface for geocoding, set UseGeocodingInAddressBook parameter. Set this parameter in each application that requires support. For example, to support the user interface in both ClaimCenter and ContactCenter, set this in both ClaimCenter and ContactCenter.
<param name="UseGeocodingInAddressBook" value="true"/>
