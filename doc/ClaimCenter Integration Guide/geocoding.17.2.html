ClaimCenter Integration Guide : Geographic Data Integration : Geocoding Plugin Integration
Geocoding Plugin Integration
Guidewire ClaimCenter and Guidewire ContactCenter provide a user interface and integration API for assigning latitude and longitude numbers to a street address (or partial address). Those numbers indicate a specific location on the Earth. This process is called geocoding. A Guidewire application requests geocoding from a registered geocoding plugin (GeocodePlugin) interface. The plugin typically would not directly geocode the address, but instead request geocoding from a remote service, such as the Microsoft MapPoint service. If the external service supports driving directions and maps, the geocoding plugin can also retrieve that information and return it the Guidewire application for storage and display.
Guidewire applications use this geocoding information for actions within rules, or for other geographic searches. The default configuration supports these high-level features:
•
ClaimCenter user assignment and searching. ClaimCenter can assign claims to users based on proximity between two addresses, such the insured’s address and user addresses. You can also search for users by proximity from the administration user search page. This feature requires ClaimCenter to have the geocoding plugin enabled and its database contain geocoded addresses
•
ContactCenter address book searches (proximity search of vendors). If both ClaimCenter and ContactCenter are installed, you can search for nearby service providers in the address book based on geographic proximity. This feature requires:
•
ClaimCenter and ContactCenter must have the geocoding plugin enabled.
•
ContactCenter database must contain geocoded addresses.
To support user assignment and searching using geocoding, ClaimCenter must install and register a geocoding plugin. To support address book searches that use geocoding, both ClaimCenter and ContactCenter must install and register a geocoding plugin.
IMPORTANT To support address book searches with geocoding, configure and install both ClaimCenter and ContactCenter, linking the two applications. See the ClaimCenter Installation Guide for details.
The geocoding plugin performs the following tasks:
•
Assign a latitude coordinate, a longitude coordinate, and a geocode status to an address. A geocode status indicates success/failure but also indicates the meaning of the coordinates, for example, is it an exact match or the center of the postal code.
•
Fix address properties (optional).
•
Return driving directions (optional). If the external service supports driving directions, ClaimCenter can call the geocoding plugin to request driving directions, optional segment maps between two addresses, and an optional overview map.
•
Find an address from coordinates (optional). This is reverse geocoding.
•
Find maps for arbitrary addresses.
From the perspective of the application requesting geocoding, geocoding is always synchronous in the sense that the caller waits until the method completes. For user-initiated requests in the user interface, the user interface blocks until the plugin responds.
Guidewire applications also support batch asynchronous geocoding using a distributed system that allows all servers in the cluster to perform geocoding in the background while other tasks happen. This distributed work queue system is especially useful after an upgrade with new address data to geocode. Use work queue infrastructure for this background task even if there is only a single server (not a cluster) for each Guidewire application.
The default config.xml file and scheduling files are pre-configured with default settings for the work queue system. Only change those settings if the defaults are inappropriate. See the ClaimCenter System Administration Guide to configure work queues such as the geocode and abgeocode work queues. You need only to implement the plugin interface’s one geocoding method and register the plugin correctly and then let the distributed work queue system handle background geocoding automatically.
Because communications to geocoding services are initiated by ClaimCenter, a reverse proxy is not necessary to protect ClaimCenter from incoming requests from the external Internet. However, insulating ClaimCenter outgoing requests through a proxy is still a good network practice. For more information about designing proxy servers and configuring an Apache web server to act as a proxy, see "Proxy Servers".
The Built-in MapPoint Geocode Plugin
ClaimCenter includes one fully functional and supported implementation of the geocoding plugin to connect to the Microsoft MapPoint web service. For details about installing and configuring MapPoint support, see “Using the Geocoding Feature” in the System Administration Guide.
