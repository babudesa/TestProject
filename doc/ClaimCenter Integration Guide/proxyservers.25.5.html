Proxy Server Integration Types for ClaimCenter
ISO Proxy Communication
ClaimCenter and ISO exchange different types of messages:
ClaimCenter sends messages 1 and 2 to ISO. For these messages, use this configuration:
•
"Downstream Proxy With Encryption"
ISO sends Message 3 asynchronously to ClaimCenter. For these message, use this configuration:
•
"Upstream (Reverse) Proxy with Encryption for Service Connections"
Metropolitan Proxy Communication
For Metropolitcan reports, use the following configuration to connect to the Metropolitan Request URL:
•
"Downstream Proxy With Encryption"
Use the same URL later to poll for availability of results:
ClaimCenter retrieves the report (after determining availability) using the separate URL called the Metropolitan Report Retrieval URL. Use the same configuration block, but with the different URL.
Geocoding Proxy Communication (Microsoft MapPoint)
If you use the included Microsoft MapPoint geocoding plugin, ClaimCenter initiates four different types of HTTP requests to MapPoint. For these four URLs, use four configurations of:
•
"Downstream Proxy With No Encryption"
If you use Guidewire ContactCenter and geocoding is enabled within ContactCenter, typically you can reuse the existing four building blocks to communicate between ContactCenter and MapPoint. Simply allow connections from both IP addresses (ClaimCenter and ContactCenter IP addresses) for those configuration building blocks.
However, if you want the geocoding plugin to connect to the proxy server, you must tell the geocoding plugin how to connect with the proxy server rather than MapPoint. To do this, in Studio define the web service to use your proxy server URL, not the real MapPoint URL. In Studio, click Web Services, then MapPoint, and then edit the configuration. See “Working with Web Services” in the Configuration Guide for more information.
For more geocoding information, see "Geographic Data Integration".
SSL Encryption for Users
You may want to use the Apache server to handle SSL encryption from users to ClaimCenter and reduce the processing burden of SSL encryption in Java on the ClaimCenter server. Use the following Apache configuration building block:
•
"Upstream (Reverse) Proxy with Encryption for User Connections"
For more information about SSL encryption, see the ClaimCenter System Administration Guide.
