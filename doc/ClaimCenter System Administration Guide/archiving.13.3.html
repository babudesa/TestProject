ClaimCenter System Administration Guide : Archiving Claims : Configuring Claim Archiving
Configuring Claim Archiving
Guidewire supports archiving on all databases available for ClaimCenter. However, both databases must be the same type (either both SQL Server or both Oracle).
Archive Parameters in config.xml
Set these configuration parameters in config.xml:
 
Parameter
Type
Description
ArchiveEnabled
Boolean
Required. Whether archiving is enabled (set to true) or disabled (set to false). Default is false. For archiving to work, you must set ArchiveEnabled to true and configure an archive database.
Once you set ArchiveEnabled to true, the server will not start if it is subsequently set to false.
This parameter also controls the creation of indexes on the ArchivePartition column. If set to true, ClaimCenter creates a non-unique index on that column for Extractable entities.
ArchiveServer
Boolean
Optional. Whether to start this server in archive mode. Default is false.
ArchiveDatabase
String
Optional. If ArchiveServer is true, this parameter is used to tell the server which archive database to use. If this parameter is not specified, the primary archive database is used.
AssignClaimToRetriever
Boolean
Whether or not to assign the restored claim to retriever. The default value of this parameter is false. If false, restored claims are assigned to the original group and user who owned them. If it is not possible to reassign to the original user, ClaimCenter assigns it to that group’s supervisor. If it cannot be reassigned to the original group, ClaimCenter assigns it to the default owner.
DaysClosedBeforeArchive
Integer
Required. Default is 30. The number of days since an item is closed before it is eligible for archiving. This parameter is used by the Archive work queue.
RestorePattern
String
Code of the activity pattern ClaimCenter uses to create restore activities.Default is retire. Upon restoring a claim, ClaimCenter creates two activities: one for the retriever, and one for the assigned user of the claim.
The Restore Claim Activity Pattern
This pattern can be used to notify another user that a claim has been restored and assigned to that user, or for any other purpose. Create this pattern as you configure the archive feature. The pattern must contain the code defined in the RestorePattern parameter in config.xml.
Configuring the Archive Work Queue
You can modify the rules that control which claims the archive work queue archives. See “Archive” in the Rules Guide.
You can also configure how the archive writer and worker daemons behave. See "Configuring Distributed Work Queues".
See "Monitoring Claim Archiving" for instructions to see the current status of both archiving processes and to run them manually.
