Enabling JMX in ClaimCenter
ClaimCenter provides a plugin that you can use for JMX connections.
To enable the plugin
1.
Open Guidewire Studio by running gwcc studio from the ClaimCenter/bin directory.
2.
In the Resources pane, select configuration → Plugins → ManagementPlugin.
3.
Select the Enabled checkbox. Studio opens a dialog box informing you that editing the plugin creates a copy in the current module.
4.
Click Yes on the dialog box.
5.
If you are using a different RMI port for the management server than the default 1099, edit the value set to the rmiPort parameter.
6.
Click File → Save Changes.
7.
If you are using the QuickStart configuration, restart the ClaimCenter server by running gwcc dev-start from ClaimCenter/bin.
8.
If you are enabling the plugin in a production environment, repackage the ClaimCenter .ear or .war file, and redeploy the file to the application server. See “Deploying ClaimCenter to the Application Server” in the Installation Guide.
If the server starts correctly, the console log shows something similar to the following on startup:
Starting JSR 160 connector on port 1099
9.
In the JMX management console, create a JMX server connection using the following settings:
 
Setting
Value
host
The hostname of the ClaimCenter server.
JNDI name
jrmp
port
1099 (default)
IMPORTANT Check that the port you assign for the JMX server connection is not already in use. If the port is in use, specify an available port.
