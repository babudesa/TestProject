Maintaining the Database
While ClaimCenter automatically handles most changes to its schema, involve a database administrator in tuning and managing the database server, including the following activities:
•
monitoring tablespace size allocations and disk space to making sure that ClaimCenter does not run out of space
•
backing up the database periodically to support disaster recovery options
•
updating database statistics periodically so that the query optimizer selects an efficient plan for executing application queries
•
running consistency checks on the database
ClaimCenter provides tools for updating database statistics and running consistency checks. Launch these tools using the -dbcatstats and -checkdbconsistency options of the system_tools command. For information on using these tools, see "system_tools Command".
