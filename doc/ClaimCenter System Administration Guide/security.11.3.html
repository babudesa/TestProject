Key Access Control Configuration Files
By default, ClaimCenter uses access control. You can change this configuration using the following config.xml parameters:
 
Parameter
Description
UseACLPermissions
Specifies whether ClaimCenter uses access control for claims at all. If set to false, then ClaimCenter ignores the access control definitions, and all users have access to all claims. ClaimCenter still restricts the actions that a user can take by the userâ€™s roles and permissions.
This value only applies to claim access control. If you set this value to false, you can still use exposure and document access control features.
RestrictSearchesToPermittedItems
If set to true, then searches return only items that the user is able to view. If set to false, then search results might include items that the user cannot view.
EnableDownlinePermissions
If set to true, then a user has access to all claims to which any user or group they supervise or manage also has access. This applies recursively down the group hierarchy. If set to false, then a supervisor must have access that is independent of the users they supervise.
This value only applies to claim access control.
Manage access control in ClaimCenter using the following files:
 
 
File
Role it plays in access control
config.xml
Contains parameters for turning access control on or off and for configuring how it behaves.
/security/security-config.xml
Defines the access control list elements.
/extensions/ClaimAccessType.ttx
Specifies different types of access related to claims.
/extensions/ClaimSecurityType.ttx
Defines the claim security types. This typelist specifies the contents of the claim user interface Special Claim Permission drop down.
/extensions/DocumentSecurityType.ttx
Defines document types for use with access control.
/extensions/ExposureSecurityType.ttx
Defines access control types related to exposures.
/extensions/SystemPermissionType.ttx
Contains customizable and custom system permissions.
/extensions/UserRole.ttx
Defines possible claim user roles. These roles appear on the Users tab of the claim.
