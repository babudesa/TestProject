ClaimCenter System Administration Guide : Archiving Claims : Converting a Primary Archive to a Secondary Archive
Converting a Primary Archive to a Secondary Archive
A secondary archive is an archive database that was formerly a primary archive. A secondary archive only stores previously archived data. No new data is moved into a secondary archive. Data is always archived to the primary archive only. ClaimCenter searches and can restore from both primary and secondary archives. You can define any number of secondary archives.
When you upgrade ClaimCenter, you will need to also upgrade archive databases. For upgrade performance reasons,
ClaimCenter does not synchronize administrative data with secondary archive databases. Over time, the administrative data become stale. As long as a user still exists in the main database, claims assigned to that user can be restored. The user can be retired or active. However, user passwords and other administrative data are not synchronized between the main database and secondary archives. Therefore, to log in to a secondary archive in archive mode, enter the password for the user as it existed when the secondary archive was converted from a primary archive.
Before converting a primary archive database to a secondary archive, do the following:
•
Synchronize administrative data by running the archive work queue. Note that any administrative data changes after that will not be reflected in the secondary archive database.
•
Start a server in archive mode against that database. See "Using an Archive Database as the Main Database". Then do the following:
•
Run and download the data distribution. See "Data Distribution".
•
Run database statistics. See "Generating Database Statistics on Archive Databases"
•
Run any necessary reports.
WARNING The name associated with an archive database cannot change. A jdbcURL must always have the same associated name. Care must be taken to preserve the name of the database. The application keeps track of which database an entity are archived to using the database name. If the database name is changed, the server will not startup.
To convert a primary archive to a secondary archive
1.
Start Guidewire Studio by running the following command from ClaimCenter\bin:
gwcc studio
2.
In the Studio Resources pane, double-click Other Resources → config.xml.
3.
Copy the archive element within the database block.
4.
Paste the copied archive element after the closing </archive> tag.
5.
Rename the copied archive element to secondaryarchive in the opening and closing tags.
6.
Modify the original archive element to point to an empty database. This database becomes the new primary archive.
7.
Import the ClaimCenter schema to the new primary archive.
8.
Select File → Save Changes.
