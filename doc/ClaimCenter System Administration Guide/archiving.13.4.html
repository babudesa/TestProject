ClaimCenter System Administration Guide : Archiving Claims : Monitoring Claim Archiving
Monitoring Claim Archiving
ClaimCenter has three server tools to help you run and monitor the archiving batch processes:
•
Work Queue Info shows the status of the archive work queue and lets you run its writer.
•
Archive Info gives status information about the archive work queue and the archive database.
ClaimCenter also has command line methods and APIs to archive and restore claims.
Work Queue Info
To start an unscheduled run of the archive work queue, or to view the status of this (and other) distributed work queues:
•
ALT+SHIFT+T → Server Tools tab → Work Queue Info menu choice → Archive row
To start an unscheduled run of the archive work queue, click the Run Writer button for the Archive row.
A Statistics table at the bottom of this tool gives information about what the writer and worker of this work queue have done.
Archive Info
Use the Archive Info tool to view information about the archive. You must have the ArchiveEnabled parameter set to true in config.xml to view the Archive Info tool.
To access the Archive Info tool
1.
In ClaimCenter, press ALT+SHIFT+T.
2.
Select the Server Tools tab.
3.
Select the Info Pages menu choice.
4.
Select Archive Info from the Info Pages drop-down.
This page includes the following information:
•
Excluded from Archive: The number of claims previously excluded by the archive work queue. To remove the excluded flag from these claims, click Reset All. For more information on the state of the archive work queue, click View Detailed Status.
•
Archive Database: The name of the archive database. This is the value set to the name attribute of the archive element in config.xml.
•
Status: The connection status of the archive database. This must show a status of Linked, or ClaimCenter is not linked to an archive database. If the status shown is Schema Out of Date, the server is connected to the archive database, but the two databases are not linked.
•
Entities Archived: The number of claims currently in this archive database.
•
Items With Errors: A list of claims that could not be archived due to an error. Items are grouped by the Archive Failure Reason.
To refresh any of the information on the Archive Info page, click Refresh.
Note: All archiving and restoring operations also produce the usual log information.
