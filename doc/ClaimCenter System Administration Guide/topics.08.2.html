ClaimCenter System Administration Guide : Enabling JMX with ClaimCenter : Overview of JMX Management Tasks
Overview of JMX Management Tasks
ClaimCenter provides access to management features through JMX (Java Management Extensions), enabling you to monitor various aspects of ClaimCenter operation and dynamically change certain configuration settings. You can connect to the ClaimCenter JMX server using a JMX management console. ClaimCenter uses the MX4J 1.1 implementation of JMX. Through JMX you can manage the following:
•
Cache —View the cache settings for each type of ClaimCenter entity. You can also set the maximum size of the cache, and how long a cached object remains in the cache.
•
Configuration —View the values of all configuration parameters. You can also change the value of certain parameters. These changes take effect in the server immediately, without requiring the server to restart. Changes made this way are discarded when the server shuts down. Upon restart, the server uses parameter values that are specified in the config.xml configuration file.
•
Monitoring —You can view the following system information:
•
batch processes currently running (if any)
•
number of current user sessions and their user names
•
number of active and idle database connections
You can also monitor notifications, which are triggered when ClaimCenter locks a user out for excessive login failures.
For a list of the MBeans available with ClaimCenter, see "Management Beans".
