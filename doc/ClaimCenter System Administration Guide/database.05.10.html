Purging Orphaned Policies from the Database
Each claim has an associated policy record. If a user has refreshed the policy information on a claim, the old policy record remains in the database. As part of regular system maintenance, check for orphaned policies in the database.
Run the following query to determine the number of orphaned policies:
SELECT count(*) FROM cc_policy p
WHERE p.retired = 0 AND p.id NOT IN (SELECT policyid FROM cc_claim)
If you find a significant number of orphaned policies, Guidewire recommends that you remove these policies. If you find a small number of orphaned policies, you might want to retire those policies rather than remove them. Contact Guidewire Support for assistance with either operation.
