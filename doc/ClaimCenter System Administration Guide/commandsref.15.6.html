table_import Options
You can use the table_import command to load administrative data from staging tables into ClaimCenter.
WARNING You can only import administrative data from staging tables. Loading any other type of data is not supported.
You can use any of the following options with the table_import command. You must always supply the -password option. Before you can use this command, you must set the ClaimCenter server run level to MAINTENANCE. You can do this with the system_tools command. See "system_tools Command".
Guidewire provides corresponding methods on ITableImportAPI for each of the table_import command options.
For more information, including detailed procedures to import bulk administrative data from staging tables, see “Importing from Database Staging Tables” in the Integration Guide.
 
Option
Description
-D name=value
Sets a Java system property.
-allreferencesallowed
Allows references to existing rows in all source tables, including administrative tables such as users and groups.
If there are rows in staging tables for CheckGroup or CheckPortion, you must set the -allreferencesallowed flag.
-batch
Runs the command in a batch process. This flag only applies with the following:
-integritycheck
-integritycheckandload
-populateexclusion
-deleteexcluded
-updatedatabasestatistics
-clearerror
Clears the error table.
-clearexclusion
Clears the exclusion table.
-clearstaging
Clear the staging tables.
-deleteexcluded
Deletes rows from staging tables based on contents of exclusion table.
-estimateorastats
This parameter applies only to Oracle databases. It updates database statistics on the source tables with estimated row and block counts for the source tables and indexes at the beginning of load (-integritycheckandload).
-help
Displays the command usage.
-integritycheck
Validates the contents of the staging tables. You can optionally specify:
-allreferencesallowed
-clearerror
-populateexclusion
-integritycheckandload
Validates the contents of the staging tables and populate source tables. You can optionally specify one of the following flags:
-allreferencesallowed
-clearerror
-estimateorastats
-populateexclusion
-zonedataonly
-messagesinks sinks, ...
Deprecated. This flag does not do anything.
-password password
Specifies the administrative password. You must specify a password.
-populateexclusion
Populate the exclusion table with rows to exclude.
-server url
Specifies the ClaimCenter host server url.
-updatedatabasestatistics
Updates the database statistics on the staging tables. If you also specify the -integritycheckandload option, this option calculates the estimated row and block counts for the source tables. When running against Oracle, this command updates indexes before populating.
-user user
Specifies the user to run the process.
-zonedataonly
Sets the import to load zone data only. Used with the -integritycheckandload flag.
