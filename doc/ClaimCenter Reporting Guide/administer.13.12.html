Understanding InetSoft Configuration Options
There are a number of configuration options available in the InetSoft Administration functionality, many of which are critical for integration with Guidewire ClaimCenter. Modifying these parameters incorrectly can invalidate the integration between InetSoft Style Report and Guidewire ClaimCenter. The following table lists critical parameters that you must not modify, or modify only on instructions provided in this document. To access these parameters, open the Enterprise Manager and navigate to the Server tab.

License Keys
ClaimCenter references this URL in file config.xml to launch the InetSoft Report Viewer from within ClaimCenter. The ability to run the InetSoft application and access the report server requires valid license keys.
You do not need to change either of the parameters.
Server → Security
Security
User Defined Security Provider displays the following value:
  com.guidewire.inetsoft.cc.GWSecurityProvider
Do not change this value without instructions from Guidewire. Doing so breaks the user authentication process between Guidewire ClaimCenter and the InetSoft report server. See “Understanding Report Security” for details of the authentication process.

Presentation → DHTML
Report Session Timeout (text box)
Set to 60 minutes. This duplicates the session time-out value set for Guidewire ClaimCenter. Setting these two values the same minimizes the possibility of time-out errors between the two applications.
You can also set this value directly in sree.properties:
  html.session.timeout=3600000
This parameter sets the timeout period for each report viewing session (in milliseconds).

→ Windows and Frames
Page Frame Name (text box)
ClaimCenter uses the value inetframe to properly display InetSoft within ClaimCenter. Do not change this value.
